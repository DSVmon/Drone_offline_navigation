# Автономный спасательный дрон для исследования шахт (MVP)

Данный репозиторий содержит программное обеспечение для автономного спасательного дрона, предназначенного для навигации в условиях обрушившихся шахт без сигнала GPS.

## Ключевые возможности (Phase 4 MVP)
- **Visual-Inertial Odometry (VIO):** Высокоточная локализация на базе данных с RGB-камеры и IMU.
- **dToF-мониторинг:** Лазерное зондирование препятствий в условиях низкой освещенности.
- **Safety Interlock (Система безопасности):** Автоматическое экстренное торможение (Emergency Brake) и удержание позиции при обнаружении препятствий на расстоянии менее 0.5 м.
- **Контейнеризация:** Полностью настроенная среда разработки и симуляции на базе Docker.

## Архитектура системы
- **Вычислитель:** NVIDIA Jetson Orin Nano + MCU (Flight Controller).
- **Сенсорика:** RGB-камера, IMU, dToF-датчик дальности.
- **Программный стек:** ROS 2 Humble.

## Быстрый запуск (Quick Launch)

Проект оптимизирован для работы в ОС **Ubuntu 24.04** и не требует локальной установки ROS 2. Все зависимости запускаются внутри Docker-контейнера.

### Шаг 1: Подготовка окружения
Убедитесь, что у вас установлены Docker и Docker Compose (v2):
```bash
sudo apt update
sudo apt install docker.io docker-compose-v2
sudo usermod -aG docker $USER
# Перезагрузите сессию (logout/login)
```

### Шаг 2: Запуск симуляции
Склонируйте репозиторий и запустите демо-скрипт:
```bash
git clone <repository_url>
cd drone_rescue_ws
chmod +x run_demo.sh
./run_demo.sh
```

Демо-скрипт автоматически:
1. Настроит права доступа к X11 для отображения GUI (Gazebo и RViz2).
2. Соберет Docker-образ с ROS 2 Humble и всеми зависимостями.
3. Запустит симуляцию дрона в виртуальной пещере (Tunnel World).

## Структура пакетов
- `src/drone_perception`: Обработка данных с датчиков и логика обнаружения препятствий (C++).
- `src/drone_navigation`: Оболочка для VIO-алгоритмов и публикация одометрии (C++).
- `src/drone_control`: Полетный контроллер и конечный автомат состояний (Python).
- `src/drone_simulation`: Модели URDF, миры Gazebo и конфигурации запуска.

## Разработка и тестирование
Для ручного управления контейнерами используйте стандартные команды Docker Compose из корня проекта:
```bash
docker compose -f docker/docker-compose.yml build
docker compose -f docker/docker-compose.yml up
```

### Тестирование системы безопасности
Чтобы проверить работу Emergency Brake, направьте дрон к стене в симуляторе Gazebo. При достижении критического расстояния (0.5м) в терминале появится лог `[SAFETY] Obstacle detected!`, и дрон автоматически остановится.
