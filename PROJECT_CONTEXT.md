# Drone Offline Navigation: Phase 4.2 (Procedural Caves & Physical Turns)

Этот файл содержит полное описание текущего состояния проекта автономного спасательного дрона для обеспечения преемственности в новых сессиях разработки Trae.

## 🎯 Цель проекта
Разработка автономной системы навигации дрона в закрытых пространствах (пещерах) с использованием стереозрения (OpenCV) в качестве основного сенсора и лазерного dToF в качестве резервного.

## 🛠 Технический стек
- **ОС:** ROS 2 Humble
- **Симулятор:** Gazebo Classic
- **Языки:** C++ (навигация/перцепция), Python (управление)
- **Библиотеки:** OpenCV 4.x, `cv_bridge`, `image_geometry`, `message_filters`
- **Среда:** Docker-контейнер с поддержкой GUI (X11)

## 🏗 Архитектура системы (Ноды)
1. **`navigation_node` (C++):** Обрабатывает стереопару (800x600 -> 400x300), вычисляет облако точек в ROI и определяет дистанцию до стен. Публикует алерты в `/navigation_node/stereo_obstacle_front`.
2. **`perception_node` (C++):** Обрабатывает данные с лазерных дальномеров dToF и сенсора столкновений (Bumper).
3. **`control_node` (Python):** Логика поведения. Использует конечный автомат миссии (SEARCHING, TURNING, RETURNING, COMPLETED).

## 🚀 Ключевые технические решения (Phase 4.2)

### 1. Процедурная генерация мира
- **`scripts/procedural_cave.py`:** Генерирует случайную извилистую пещеру при каждом запуске. 
- Используются сегменты по 1.5м с текстурой `Gazebo/Bricks`.
- Пещера полностью закрыта (стены, потолок, тупик в конце).

### 2. Миссия "Исследование и возврат"
- Дрон больше не летает бесконечным челноком. 
- **SEARCHING:** Полет вперед до обнаружения тупика (Stereo/dToF).
- **TURNING:** Разворот на 180 градусов.
- **RETURNING:** Полет обратно к точке старта (X=0).
- **COMPLETED:** Полная остановка при возвращении.

### 3. Физический разворот и навигация
- **P-контроллер:** Плавное замедление вращения при приближении к целевому углу (точность ~2.8°).
- **Yaw Extraction:** Извлечение угла рыскания из одометрии для контроля поворота.

### 3. Оптимизация и Логирование
- **Concise Logs:** В консоль выводится только критическая информация: позиция, алерты стерео, статус разворота и столкновения.
- **Collision Detection:** Красный лог `!!! COLLISION DETECTED !!!` при физическом контакте со стеной.
- **Visuals:** Дрон окрашен в ярко-оранжевый цвет для лучшей видимости в Gazемо. Включен `imshow` для отладки стерео и диспаратности.

## ⚠️ Решенные проблемы (Lessons Learned)
- **Stereo Threshold:** Порог срабатывания снижен до **0.5 м** для более плотной навигации.
- **Turn Precision:** Внедрена минимальная скорость вращения и уменьшен допуск (0.05 рад) для исключения недокрутов.
- **X11 Bottleneck:** Сообщения `Message Filter dropping message` в RViz2 связаны с нагрузкой на графический канал Docker-Windows.
