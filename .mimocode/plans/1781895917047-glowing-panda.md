## Диагноз: почему текущее обучение не работает

### Критические баги (ломают обучение на 100%)

**1. Observation dimension mismatch в inference_node.py**
- `drone_env.py:196-216` строит **19-dim** вектор: `prev_stereo[5] + stereo[5] + x, y, z, sin_yaw, cos_yaw, vx, vz, roll, pitch`
- `inference_node.py:99-113` строит **13-dim** вектор: `stereo[5] + x, y, z, sin_yaw, cos_yaw, vx, roll, pitch` — **нет prev_stereo и vz**
- PPO модель обучена на 19-dim, inference подаёт 13-dim → **полный крах инференса**

**2. Обучение падает на 3-м эпизоде (training_log.csv)**
- Эпизод 1: 163 шага, -133.39, out_of_bounds
- Эпизод 2: 132 шага, -368.06, out_of_bounds
- Эпизод 3: 86 шага, -222.31, out_of_bounds
- PPO policy сразу уводит дрон за границы → BC warm-start не спасает

**3. Expert data смешанного формата**
- 500 батчей, 100K сэмплов
- Первые ~250 батчей (26.05): **13-dim** observations
- Вторые ~250 батчей (27.05): **19-dim** observations
- `bc_model.py:load_expert_data()` фильтрует по max dim → использует только 19-dim (~50K)
- Но BC модель обучается на 19-dim, а потом权重 transfer в PPO происходит корректно

### Структурные проблемы (мешают обучению)

**4. Z-axis телепорт обходит физику**
- `drone_env.py:242-277`: vz применяется как телепорт через SetEntityState, а не как сила
- Policy не учит реальную вертикальную динамику
- `hover_correction` (строка 256) дополнительно искажает поведение — подтягивает к DRONE_SPAWN_Z

**5. Награда слишком слабая для навигации**
- `R_SURVIVE = +0.005` за шаг — слишком мало
- `R_SPEED_COEFF = 0.02 * vx` — при vx=0.8 даёт +0.016 за шаг
- Суммарно ~0.021 за шаг → за 1000 шагов = +21.0
- Но `R_COLLISION = -15.0` и `R_OUT_OF_BOUNDS = -15.0` — дрон быстро теряет награду
- **Нет.reward за продвижение вперёд по туннелю** (только за скорость)

**6. Episode timeout слишком длинный (300с)**
- Дрон может лететь 300с, набирать -5 за timeout
- Лучше сократить до 60-120с для ускорения обучения

**7. Слишком малый network для сложной задачи**
- `[128, 128]` с Tanh — 19K параметров
- Задача навигации в 100м пещере с 20-фазным зондированием требует большей ёмкости

### Этап 3: Полная переработка reward function (КЛЮЧЕВОЙ)

**3.1. Стратегия drone при обучении**

Drone учится ТОЛЬКО в режиме SEARCHING (без INSPECTING). Это радикально упрощает задачу.

**Поведение drone (целевое):**
```
Если стена дальше 1.5м:
    Лети вперёд на полной скорости (0.8 м/с)

Если стена 0.5-1.5м:
    Поверни в сторону с бо́льшим просветом
    Замедлись до 0.4 м/с

Если стена ближе 0.5м:
    Резко поверни от стены
    Замедлись до 0.2 м/с
```

**Это чисто реактивное поведение** — RL должен его освоить за 500K-1M шагов.

**3.2. Анализ текущих проблем с наградами**

| Компонент | Текущее | Проблема | Итог |
|-----------|---------|----------|------|
| R_SURVIVE | +0.005 | Слишком мало | Drone не ценит выживание |
| R_SPEED | +0.02×vx | Недостаточно | Слабый стимул к движению |
| R_PROXIMITY | -0.3×(1-d)²×5 | **Слишком сильно** | -0.375/шаг → drone боится лететь |
| R_APPROACH | -2.0×Δd | Слишком сильно | Ещё усиливает страх |
| R_PROGRESS | **НЕТ** | **Критично** | Нет сигнала "лети вперёд" |
| R_COLLISION | -15.0 | Слишком драматично | Drone боится ошибаться |
| R_OUT_OF_BOUNDS | -15.0 | Слишком драматично | Drone боится исследовать |

**Баланс текущих наград**:
- Положительная: +0.021/шаг (survive + speed)
- Отрицательная при близости: **-0.375/шаг** (5 каналов × proximity)
- **Отношение: 1:18** — штрафы в 18 раз сильнее наград!
- **Результат**: оптимальная стратегия — стоять на месте (0 reward) или лететь за границы (-5 timeout vs -15 collision)

**3.3. Новая система наград (с учётом curriculum)**

```
┌─────────────────────────────────────────────────────────┐
│           НОВАЯ СИСТЕМА НАГРАД                          │
│                                                         │
│  ПРОДВИЖЕНИЕ (доминирующий сигнал):                     │
│    r_progress = PROGRESS_COEF × max(0, Δx_along_cave)  │
│    → КЛЮЧЕВОЙ КОМПОНЕНТ                                 │
│    → Стадия 1: +1.0  (сильный стимул)                  │
│    → Стадия 2: +0.7  (умеренный)                       │
│    → Стадия 3: +0.5  (сбалансированный)                │
│                                                         │
│  ВЫЖИВАНИЕ:                                             │
│    r_survive = +0.01/шаг (постоянный)                   │
│                                                         │
│  СКОРОСТЬ:                                              │
│    r_speed = +0.03 × vx                                 │
│                                                         │
│  ОПАСНОСТЬ (пропорциональная):                          │
│    r_proximity = PROX_COEF × Σ(1-d)² при d < 0.5м      │
│    → Порог 0.5м (а не 1.0м) — drone может лететь близко │
│    → Стадия 1: -0.05  (мягко)                          │
│    → Стадия 2: -0.10  (умеренно)                       │
│    → Стадия 3: -0.15  (строго)                         │
│                                                         │
│  Сближение:                                             │
│    r_approach = -0.2 × max(0, Δd) при d < 0.5м         │
│                                                         │
│  ЦЕНТРИРОВАНИЕ:                                         │
│    r_center = +0.01 × (1 - |L-R|/max(L,R))             │
│                                                         │
│  ТЕРМИНАЛЬНЫЕ:                                          │
│    r_collision = -5.0    (terminate)                    │
│    r_out_of_bounds = -10.0 (terminate)                  │
│    r_timeout = -3.0      (terminate)                    │
│    r_stuck = -5.0        (terminate)                    │
│    r_completion = +50.0  (terminate)                    │
└─────────────────────────────────────────────────────────┘
```

```python
class CurriculumManager:
    STAGES = [
        {"name": "straight", "cave_script": "straight_cave.py", "reward_progress": 1.0},
        {"name": "gentle", "cave_script": "gentle_cave.py", "reward_progress": 0.5},
        {"name": "full", "cave_script": "procedural_cave.py", "reward_progress": 0.5},
    ]
    
    def __init__(self):
        self.current_stage = 0
        self.stage_steps = 0
        self.success_count = 0
        self.total_count = 0
    
    def should_advance(self):
        if self.stage_steps >= 500_000 and self.success_rate() > 0.8:
            return True
        return False
    
    def success_rate(self):
        if self.total_count < 10:
            return 0.0
        return self.success_count / self.total_count
```
