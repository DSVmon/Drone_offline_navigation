## 1. DEPTH MAP PROCESSING

### MAVRL (depth_callback)
```python
depth = self.bridge.imgmsg_to_cv2(msg, desired_encoding='passthrough') / 1000.0  # mm → m
depth = (np.minimum(depth, 12.0)) / 12.0 * 255.0  # [0,255]
shape = depth.shape
depth = depth[:, int((shape[1]-shape[0])/2 - 1) : int((shape[1]+shape[0])/2 - 1)]  # center crop
depth = cv2.resize(depth, (256, 256), interpolation=cv2.INTER_AREA)
self.depth = depth.astype('int')  # → int (не uint8!)
```

### Наш проект (_depth_callback)
```python
depth = self.bridge.imgmsg_to_cv2(msg, 'passthrough')  # uint8 [0,255]
# Resize если нужно
with self._lock:
    self.depth_image = depth
```

### Сравнение

| Шаг | MAVRL | Наш проект | Различие |
|-----|-------|------------|----------|
| Формат входа | mm (float32) | uint8 [0,255] | ✅ Разный формат |
| Конвертация mm→m | `/1000.0` | Нет (уже uint8) | ✅ OK |
| Clamp | `min(depth, 12.0)` | Нет | ⚠️ |
| Normalize | `/12.0 * 255.0` | Нет (уже [0,255]) | ✅ OK |
| Center crop | `depth[:, (h-w)/2:(h+w)/2]` | Нет | ⚠️ |
| Resize | `cv2.resize(256,256, INTER_AREA)` | `cv2.resize(256,256)` | ⚠️ Interpolation |
| Final dtype | `int` | `uint8` | ✅ Совместимы |

### Критичность различий

1. **Формат входа (mm vs uint8):** OK — наш navigation_node уже конвертирует в uint8
2. **Clamp:** Маловероятно что depth > 12m в пещере — OK
3. **Center crop:** Наш depth map уже square (640×480 → crop/resize). В navigation_node.cpp уже есть center crop. OK
4. **Interpolation:** INTER_AREA лучше для downsampling. Наш код использует cv::INTER_AREA в C++. OK
5. **dtype int vs uint8:** Совместимы для CNN (оба конвертируются в float32). OK

## 3. world2body TRANSFORMATION

### MAVRL
```python
def world2body(self, world):
    rot = R.from_quat(self.quat)
    world_rfu = np.array([world[1], -world[0], world[2]])
    body_rfu = np.array(rot.inv().apply(world_rfu))
    return body_rfu  # Возвращает FLU (не RFU!)
```

### Наш проект
```python
def _world2body(self, world_vel):
    quat = [0.0, 0.0, math.sin(self.current_yaw / 2.0), math.cos(self.current_yaw / 2.0)]
    rot = Rotation.from_quat(quat)
    world_flu = np.array([world_vel[1], -world_vel[0], world_vel[2]])
    body_flu = rot.inv().apply(world_flu)
    return np.array([body_flu[1], -body_flu[0], body_flu[2]])  # Возвращает RFU
```

### Сравнение

| Аспект | MAVRL | Наш проект | Различие |
|--------|-------|------------|----------|
| Quaternion | Полный (из odometry) | Только yaw | ⚠️ |
| Input format | world RFU | world RFU | ✅ |
| Rotation | `rot.inv().apply(world_rfu)` | `rot.inv().apply(world_flu)` | ⚠️ |
| Output format | body FLU | body RFU | ⚠️ |

### Критичность
**Средняя.** Различия в quaternion (полный vs yaw-only) и формате вывода (FLU vs RFU). Однако:
- Quaternion влияет только при наклоне drone (roll/pitch)
- Output format компенсируется в calRLState

## 4. ACTION APPLICATION

### MAVRL (RobotState.step)
```python
def step(self, input, duration):
    cmd = input * self.act_std + self.act_mean
    self.acc = cmd[:3]
    self.yaw_rate = cmd[3]
    acc_world = self.body2world(self.acc)
    self.vel_world = self.vel + acc_world * duration
    self.vel = self.vel_world
```

### Сравнение

| Аспект | MAVRL | Наш проект | Различие |
|--------|-------|------------|----------|
| Denormalize | `input * act_std + act_mean` | `action_arr * ACTION_STD + ACTION_MEAN` | ✅ Идентично |
| Integration | `vel = vel + acc_world * duration` | `vel_world += acc_world * DT` | ✅ Идентично |
