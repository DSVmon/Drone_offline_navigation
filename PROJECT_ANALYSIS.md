# 📊 Детальный анализ проекта Drone_offline_navigation

## 🎯 Обзор проекта

**Название:** Drone Offline Navigation (Phase 4.2: Procedural Caves & Physical Turns)

**Цель:** Разработка полностью автономной системы навигации спасательного дрона для исследования закрытых пространств (пещер, шахт) без GPS. Система использует стереозрение как основной сенсор и лазерный dToF в качестве резервного.

**Статус:** ✅ Phase 4.2 - Готово к внедрению SLAM (Phase 5)

---

## 🚀 Ключевые технические решения

### 1️⃣ Процедурная генерация пещеры

**Файл:** `scripts/procedural_cave.py`

**Особенности:**
- ✅ Генерирует уникальную пещеру при каждом запуске
- ✅ Параметры: длина ~40м, ширина 2.5-6м, высота 0.8-3.5м
- ✅ 5 экстремальных зон для проверки 3D маневрирования:
  - Узкое место (1.0м ширина)
  - Широкое место (10.0м ширина)
  - Низкий потолок (0.8м)
  - Высокий пол (ступенька 1.5м)
- ✅ Препятствия: Сталактиты/сталагмиты (Gazebo/Wood блоки)
- ✅ Полностью замкнутое пространство (стены, потолок, тупик)

**Результат:** SDF файл с процедурно сгенерированным миром

---

### 2️⃣ Стереозрение (Navigation Node - C++)

**Файл:** `src/drone_navigation/src/navigation_node.cpp`

**Технология:** OpenCV StereoBM (Block Matching)

**Алгоритм:**
```
Левое изображение  →┐
Правое изображение →┼→ StereoBM →  Disparity Map → Depth Map
                   ┴→ Ректификация
```

**Параметры BM:**
- `num_disparities = 64` - Диапазон поиска
- `block_size = 21` - Размер блока для сопоставления
- `TextureThreshold = 10` - Фильтрация низкотекстурных областей

**Выходные данные (5 зон):**
```
┌─────────────────┐
│    TOP (↑)      │  dist_top
├────┬─────┬──────┤
│L(←)│CENTER│R(→) │  dist_left, dist_center, dist_right
├────┴─────┴──────┤
│   BOTTOM (↓)    │  dist_bottom
└─────────────────┘
```

**Проблема решена:** "Stereo Blindness" (ложные 10м при подлете < 0.3м) → Решено через dToF override

---

### 3️⃣ Гибридная 3D физика (Control Node - Python)

**Файл:** `src/drone_control/drone_control/control_node.py`

**Проблема:** 
- Плагин `planar_move` не поддерживает ось Z
- Плагин `force_based_move` отсутствует в ROS 2 Humble

**Решение (Hybrid Approach):**
```python
# X, Y, Yaw движение через planar_move (стабильно)
twist.linear.x = cmd_vx
twist.angular.z = cmd_yaw

# Z движение через Gazebo SetEntityState сервис (прямое управление позицией)
entity_state = EntityState()
entity_state.pose.position.z = self.current_z + dz
self.gz_set_state_client.call_async(SetEntityStateRequest(entity_state))
```

**Преимущества:**
- ✅ Полная 3D маневренность
- ✅ Облет препятствий сверху/снизу
- ✅ Минимальные задержки

---

### 4️⃣ Конечный автомат (FSM) миссии

**Граф состояний:**
```
        ┌──────────────────┐
        │    SEARCHING     │ (Поиск пути вглубь пещеры)
        │  dist > 0.8м     │
        └────────┬─────────┘
                 │ dist < 0.8м (препятствие обнаружено)
        ┌────────▼─────────┐
        │   INSPECTING     │ (Анализ сложного препятствия)
        │ dist > 1.5м ИЛИ  │
        │ находен проход    │
        └────────┬─────────┘
                 │ (нет пути / тупик)
        ┌────────▼─────────┐
        │     TURNING      │ (Разворот 180°)
        │  reached YAW+π   │
        └────────┬─────────┘
                 │ Разворот завершен
        ┌────────▼─────────┐
        │    RETURNING     │ (Полет к входу)
        │  dist_home < 5м  │ → Pure Homing (прямо на (0,0))
        │  else            │ → Реактивное уклонение
        └────────┬─────────┘
                 │ Достигли входа (Virtual Wall)
        ┌────────▼─────────┐
        │     TURNING      │ (Разворот у входа 180°)
        └────────┬─────────┘
                 │ Перед входом
        └────────► SEARCHING (новый цикл) ◄───┐
                                              │
                    ┌──────────────────────────┘
                    │ (циклически, без остановки)
```

**Механизмы защиты:**
- ⏱️ `wall_hit_guard` - Предотвращает зацикливание на виртуальной стене
- 🔄 `previous_mission_state` - Сохраняет контекст после облета препятствия
- 📐 `entrance_heading` - Направление входа (hard virtual wall)

---

### 5️⃣ Стабильность и фильтрация

**Фильтрация сенсоров (EMA):**
```python
alpha = 0.5  # Экспоненциальное скользящее среднее
filtered_value = alpha * new_value + (1 - alpha) * previous_value
```

**Улучшения (Phase 4.2.1):**
- ✅ Вес центрального сенсора: 80% (снижает ложные торможения)
- ✅ Минимальная скорость: 0.25 м/с (уверенное движение)
- ✅ Гистерезис состояний: Вход в INSPECTING на 0.8м, выход на 1.5м

**Порог обнаружения столкновения:**
- Бампер дрона (ContactsState)
- Экстренный маневр отскока

## ⚠️ Решенные проблемы (Phase 4)

| Проблема | Решение | Результат |
|----------|---------|-----------|
| **Stereo Blindness** | dToF override при dist < 0.3м | ✅ Надежное обнаружение |
| **Frozen Z** | Гибридный метод (planar + Gazebo Svc) | ✅ Полная 3D маневренность |
| **Message Filter Dropping** | Docker → WSL2 (вGPU) | ✅ 0 пропусков кадров |
| **Sensor Jumps** | EMA фильтрация (alpha=0.5) | ✅ Плавное движение |
| **Infinite Wall Spin** | wall_hit_guard + проверка улета | ✅ Один цикл на вход |
| **Turning Precision** | Допуск 0.1 рад (жесче) | ✅ Точный разворот |
| **Return Jerking** | previous_mission_state контекст | ✅ Плавный возврат |
